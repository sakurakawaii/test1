Tariq, James,

I've pulled the gateway configuration export Tariq requested.

The relevant parameter is `delivery_retry_max_attempts`, currently set to **3**.

A couple of notes on interpreting this:

This parameter controls how many retry attempts are made after an initial delivery attempt
fails. So the sequence is: initial attempt → up to 3 retries → DLQ if all 3 retries fail.
That's 4 total delivery attempts before DLQ if you count the initial one, or 3 if you
count only the retries.

I mention this because the spec language ("5 consecutive failed delivery attempts") and the
release notes language ("3 consecutive failures") may be describing the same or different
things depending on whether "attempts" includes the initial delivery in the count. I'm not
in a position to say which interpretation was intended when each document was written.

What I can tell you is that `delivery_retry_max_attempts = 3` is the current live value.

Nadia
