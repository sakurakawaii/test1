All,

A few things I want to get in front of people before the end of the month.

**1. DLQ threshold adjustment**
Following analysis of delivery failure patterns since launch, we're aligning the gateway
configuration with the v2.1 release notes. The DLQ threshold will be set to 3 consecutive
delivery failures effective 2026-08-05. This matches the correction we published in the
release notes. Teams with alerting tuned to a 5-failure assumption should adjust.

**2. Auth credential expiry timeline**
Reminder that OAuth2 tokens issued for EventBridge use have a 3600-second lifetime. We've
had a few tickets about stale tokens — make sure your service is refreshing with at least a
5-minute buffer. The pitaron-auth-client (v2.4+) handles this automatically.

**3. Migration tracker update request**
Sofia — can you flag which rows in the tracker need the most urgent update?

**4. Staging environment availability**
The EventBridge staging environment will be in a maintenance window 2026-08-01 from
02:00–06:00 UTC. Plan around this if you're running integration tests that week.

**5. November 1 date**
No such date has been agreed by Platform Engineering. The published removal date is
December 31. I'll get a note out that clarifies this.

Connor
