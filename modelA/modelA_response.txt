Overview
The project models a B2B billing and product-usage platform: raw billing data (accounts, subscriptions, invoices) and product analytics data (usage events, feature flags) are staged and transformed into intermediate MRR, payment, usage, and account-health logic. These feed revenue marts (invoices, monthly MRR, churn events, account/plan dimensions) and product marts (feature adoption, daily usage, account health scores) for reporting.

Issues
Models with Inefficient SQL Code
* models/marts/product/fct_feature_adoption.sql — The newly added legacy_flags CTE is defined but never referenced anywhere in the model. It is dead code: Snowflake prunes unreferenced CTEs so it carries no compute cost, but it clutters the model and misleads reviewers. Recommendation: remove the CTE, or join/filter on it if legacy-flag exclusion was the intent of the change.
* models/intermediate/int_account_usage.sql — The deduped_sessions CTE (carried into this sprint's modified version) is defined but never referenced by the downstream CTEs. Recommendation: remove the unused CTE or wire it into account_flags if deduplication was intended.
Models with Incorrect Syntax
All three errors below pass dbt compile (Jinja is valid) but cause a runtime failure when dbt run executes the statement — Snowflake rejects each with an "Unknown function" SQL compilation error.
* models/intermediate/int_subscription_mrr.sql — AGE(CURRENT_DATE(), start_date) is a PostgreSQL function and does not exist in Snowflake. Correction: use DATEDIFF(day, start_date, CURRENT_DATE()) as tenure_days.
* models/marts/revenue/fct_mrr_monthly.sql — ISNULL(m.mrr_amount * fx.rate_to_usd, 0) uses the SQL Server ISNULL function, which does not exist in Snowflake. Correction: use COALESCE(m.mrr_amount * fx.rate_to_usd, 0) (or IFNULL/ZEROIFNULL).
* models/intermediate/int_account_usage.sql — The newly added first_event_type column calls the first_touch_feature macro, which expands to ELEMENT_AT(event_type_array, 1). ELEMENT_AT is a Spark/Trino function and does not exist in Snowflake. Correction: update the macro to use GET({{ array_col }}, 0) or {{ array_col }}[0] (Snowflake arrays are 0-indexed).
Models with Missing References
* models/marts/revenue/fct_mrr_monthly.sql — Missing reference: ref('int_currency_rates'). No model, seed, or snapshot named int_currency_rates exists in the project. This is a compilation failure: dbt cannot resolve the node ("depends on a node named 'int_currency_rates' which was not found"), which blocks compilation of the entire project until the model is created or the ref is removed.
* models/marts/revenue/fct_invoices.sql — Missing reference: column days_late from int_invoice_payments. The upstream model renamed the column to days_overdue this sprint, so i.days_late no longer resolves. This is a runtime failure when dbt run builds the model (Snowflake error: invalid identifier 'I.DAYS_LATE'). Fix by selecting i.days_overdue (optionally aliased back to days_late if downstream consumers depend on the old name).
Sources That Are Never Referenced in a Model
* product.feature_usage_raw (defined in models/staging/_sources_product.yml) — Ghost source: declared but never referenced via {{ source() }} in any model. Described as a legacy pre-2024 extract; this is carried over from the prior sprint rather than introduced this cycle. Recommend removing the definition or building a staging model on it.
* All other declared sources (billing.accounts, billing.subscriptions, billing.invoices, product.usage_events, product.feature_flags) are referenced by staging models.
